# 什么是 TS?

  • TypeScript 是 JavaScript 的一个超集,所谓超集，可以从以下两个方面来理解。
  1. JS 中可以实现的事情，在 TS 实现中依旧可以实现。
  2. 在 JS 中没有的一些东西，在 TS 中存在。或者可以理解为 TS 是为了 JS 以后要实现的一些事。
  • TS 可以通过一些插件实现无缝的转换成 JS，至于为什么需要转换，因为本质上浏览器是不可以直接解析 TS 的，浏览器能够识别的只有 HTML，CSS，JS。

# 为什么要使用 TS?

其实从字面文字上已经可以找到答案,TypeScript 的核心就是 Type，TS 中最主要的就是强调了类型。我们知道 JS 其实是一门弱类型的语言，对类型的概念非常不明显。

let str = "我是字符串";
str = 123;
//初始变量Str的类型为字符串,后续直接变成了数字
console.log(str);
1
2
3
4
  • 上边的代码如果是做过后端的朋友，应该知道这些在后端语言譬如 JAVA，C，C++中根本是不可能的事，程序会直接报错

当然会有人觉得 JS 已经这么方便了，为什么还要强制使用强类型的 TS 来写代码呢，不是多此一举？可能结合实际生活更好理解一些。

假如你家里有一个杯子，理论上这个杯子里可以放任何东西，可以装水，装饮料，装果汁，甚至刷牙洗漱也可以用这个杯子等等。但是仔细想想，这样好像并不健康。

所以在 TS 中强调的就是分类，并指定当前容器具体是做什么。这样对于代码管理，整个当前应用的生态都会有比较好的效果。

# 安装 TS

  1. 命令行运行如下命令，全局安装 TypeScript：
npm install -g typescript
1
  1. 安装完成后，在控制台运行如下命令，检查安装是否成功(4.x)：
tsc -V
1

# 小试牛刀

  1. 新建一个 demo.ts 文件。写一些简单代码
let str: string = "我是前端菜鸡";
console.log(str);
1
2
  1. 使用 tcs 命令 转换为 js 文件.
tsc demo.ts

# 使用配置文件

tsc --init

提示

运行命令后会自动生成 tsconfig.json 文件，之后只需要运行 tsc(后续不再跟文件名)，会自动把当前目录下所有 TS 文件转为 JS。
